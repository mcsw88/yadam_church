# GStack Feature Builder (yedam)

기능 작업을 **계획·단계화**한다. GStack 세부(Y/N, office-hours, 단계 순서)는 **`gstack-workflow.mdc`** 를 따른다.

## 단계

1. `AGENTS.md`, `TASK.md`, `WORKFLOW_GUIDE.md` 확인
2. 목표·비목표·완료 정의 작성
3. **필요 시** 사용자에게 알린다: GStack 워크플로는 `gstack-workflow.mdc` 준수 (여기서 단계를 다시 쓰지 않음)
4. 구현 계획(파일 단위)을 초안 — 승인 후 실행
5. 제안·정책 변경은 `PROPOSAL_INBOX.md`·`06-proposal-integration` 절차

## 게이트

- 승인 없이 다수 파일 수정하지 않음
