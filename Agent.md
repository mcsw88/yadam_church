# AGENTS.md

## 프로젝트 목적

이 프로젝트는 교회 웹사이트로서의 차분함, 신뢰감, 정돈된 분위기를 유지하면서  
현재 메인화면에 형성된 미국/유럽식 미니멀 교회 사이트 느낌을 기준으로  
모든 페이지의 구조, 색상, 폰트, 여백, 메뉴 체계를 일관되게 확장하는 것을 목표로 한다.

이 프로젝트는 한국형 교회 사이트처럼 배너, 박스, 색상, 정보가 과하게 많은 구조를 지향하지 않는다.  
핵심은 넓은 여백, 얇은 구분선, 절제된 색상, 고정된 폰트 위계, 단정한 레이아웃이다.

---

## 절대 금지

- 스타트업 랜딩페이지 느낌 금지
- 한국형 포털/배너형 교회 사이트 느낌 금지
- 과한 카드 UI 금지
- 과한 그림자, 과한 애니메이션 금지
- 요청받지 않은 색상 체계 변경 금지
- 요청받지 않은 폰트 추가 금지
- 현재 메인화면에 없는 새 색상 임의 추가 금지
- 페이지마다 다른 디자인 톤 생성 금지
- 전체 페이지를 한 번에 대규모 수정 금지
- 레거시 HTML을 실사용 기준으로 삼는 것 금지

---

## 여백 원칙

- 여백은 이 사이트의 핵심 디자인 요소다.
- 현재 메인화면의 넓고 차분한 호흡감은 유지한다.
- 단, 의미 없는 빈 공간이나 모바일에서 흐름을 끊는 과도한 여백은 피한다.
- 섹션 간격, 제목과 본문 사이 간격, 소메뉴와 본문 사이 간격은 메인화면 기준으로 통일한다.
- 정보를 억지로 많이 넣기 위해 여백을 줄이지 않는다.
- 내용이 적은 페이지는 여백으로 정돈하고, 내용이 많은 페이지는 섹션으로 나눈다.

---

## 작업 원칙

- 작업 전 현재 구조를 먼저 분석한다.
- 수정 계획을 먼저 제시한다.
- 승인 없이 바로 수정하지 않는다.
- 한 번에 한 영역 또는 한 기능 단위만 수정한다.
- 불확실하면 추정하지 말고 확인한다.
- 완료 후 변경 파일과 이유를 요약한다.
- 한 번의 변경 범위는 작게 유지한다.
- 가능하면 한 번의 코드 변경은 500줄 이하로 유지한다.

---

## 반응형 원칙

- 고정폭 남발 금지
- px 남발 금지
- 모바일 메뉴 겹침 금지
- 모바일에서는 hover를 전제로 하지 않는다.
- 모바일 메뉴는 햄버거 또는 accordion 구조를 우선한다.
- 터치 영역은 충분히 확보한다.
- PC와 모바일 모두에서 여백과 메뉴 구조가 자연스러워야 한다.

---

## 디자인 시스템 원칙

- 현재 메인화면의 스타일을 원본 디자인 시스템으로 본다.
- 색상은 현재 사용 중인 색상만 사용한다.
- 폰트는 현재 사용 중인 폰트만 사용한다.
- 여백은 현재 메인화면 기준으로 통일한다.
- Header, PageHeader, SubNavigation, MainContent, Footer 구조를 일관되게 유지한다.
- 대메뉴와 소메뉴는 익숙한 웹사이트 구조를 따르되, 미니멀하게 구현한다.
- SubNavigation은 필요할 경우 PageHeader 아래, MainContent 위에 배치한다.
- 소메뉴는 text-link, underline tab, minimal pill 수준으로만 표현한다.
- 강한 색상의 탭, 두꺼운 박스형 메뉴, 과한 border, 과한 shadow는 금지한다.

## 검증 원칙

- 가능한 경우 미리보기로 데스크톱과 모바일을 모두 확인한다.
- Header, dropdown, mobile menu, SubNavigation이 겹치지 않는지 확인한다.
- 메인화면의 시각적 결과가 깨지지 않았는지 확인한다.
- 새 페이지가 메인화면과 같은 사이트처럼 보이는지 확인한다.
- 승인 기준을 만족하지 못하면 완료로 간주하지 않는다.

2. 현재 구조 분석 요약

4. 수정 계획

7. 승인 후 진행할 작업 순서
